# Smart Showcase

Личный кабинет продавца с AI-ассистентом для улучшения объявлений. Продавец видит список своих объявлений, может просматривать и редактировать их, а также получать рекомендации от нейросети — генерацию описания и анализ рыночной цены.

## Стек технологий

**Клиент:**
- React 18 + TypeScript
- Vite
- react-router-dom v6
- SCSS Modules
- Кастомные хуки вместо react-query (`useFetch`, `useDebounce`, `useLocalStorage`, `useAI`)
- Нативный `fetch` с `AbortController`

**Сервер:**
- Node.js + Express + TypeScript
- ts-node-dev
- JSON файл как база данных
- Groq API (llama-3.3-70b-versatile) для AI функций

## Запуск проекта

### Требования
- Node.js v20+
- Аккаунт на [console.groq.com](https://console.groq.com) для AI функций

### 1. Клонируй репозиторий

```bash
git clone https://github.com/твой-username/smart-showcase.git
cd smart-showcase
```

### 2. Настройка сервера

```bash
cd server
npm install
```

Создай файл `server/.env`:

```
PORT=3001
GROQ_API_KEY=твой_ключ_от_groq
```

Получить ключ Groq: зайди на [console.groq.com/keys](https://console.groq.com/keys) → Create API Key.

Запусти сервер:

```bash
npm run dev
```

Сервер запустится на `http://localhost:3001`

### 3. Настройка клиента

```bash
cd ../client
npm install
npm run dev
```

Клиент запустится на `http://localhost:5173`

## API эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/items` | Список объявлений с фильтрами, поиском, сортировкой, пагинацией |
| GET | `/items/:id` | Одно объявление |
| PUT | `/items/:id` | Обновление объявления |
| POST | `/ai/suggest-description` | Генерация описания через AI |
| POST | `/ai/suggest-price` | Анализ рыночной цены через AI |

### Параметры GET /items

| Параметр | Тип | Описание |
|----------|-----|----------|
| `q` | string | Поиск по названию |
| `limit` | number | Количество на странице |
| `skip` | number | Пропустить N записей |
| `categories` | string | Категории через запятую: `auto,electronics` |
| `needsRevision` | boolean | Только требующие доработок |
| `sortColumn` | string | Поле сортировки: `title` или `createdAt` |
| `sortDirection` | string | Направление: `asc` или `desc` |

## Функциональность

- **Список объявлений** — поиск, фильтры по категории, переключатель "Только требующие доработок", сортировка, пагинация с окном из 5 страниц
- **Просмотр объявления** — все характеристики, блок незаполненных полей
- **Редактирование** — динамические поля в зависимости от категории, валидация обязательных полей, сохранение черновика в localStorage, toast уведомления
- **AI ассистент** — генерация описания и анализ рыночной цены через Groq API, результат в тултипе с кнопками "Применить" и "Закрыть"

## Принятые решения

**Без внешних библиотек для стейта и запросов** — вместо Redux и React Query написаны кастомные хуки. Это позволило лучше понять как работают эти инструменты внутри.

**JSON файл как БД** — для простоты разработки. В продакшене заменить на PostgreSQL или MongoDB.

**Прокси через Vite** — все запросы к `/api/*` проксируются на сервер. Ключ Groq API хранится только на сервере и никогда не попадает на клиент.

**Черновик в localStorage** — при обновлении страницы форма восстанавливается. Черновик удаляется после сохранения или отмены.

**AbortController** — при переходе между страницами незавершённые запросы отменяются чтобы избежать обновления размонтированных компонентов.
